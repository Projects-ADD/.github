Portafolio técnico de **<!-- [NOMBRE O ALIAS] -->**, desarrollador Backend con orientación a **Arquitectura de Sistemas y Software Jr.**, enfocado en el diseño de soluciones escalables, seguras e inteligentes.

## 🗂️ Principales enfoques

Los repositorios están organizados en equipos dentro de esta organización, agrupados por área de especialización:

| Área | Descripción |
|------|-------------|
| ⚙️ **Backend & APIs** | Servicios REST / GraphQL, autenticación, integración entre sistemas |

## 📐 Filosofía de Desarrollo

| Principio | Enfoque |
|-----------|---------|
| 🧱 Clean Architecture | Separación de responsabilidades y código mantenible |
| 📏 SOLID & DRY | Diseño orientado a extensibilidad y bajo acoplamiento |
| 🔒 Security by Design | La seguridad como parte del proceso, no un añadido |
| 📄 API-First Design | Contratos claros antes de la implementación |
| 📚 Docs como código | Documentación viva junto al repositorio |
| 🤝 Colaboración abierta | Proyectos abiertos a contribuidores y revisión de pares |

## 📋 Marcos Normativos de Referencia

| Dominio | Norma | Aplicación |
|---|---|---|
| Calidad de Software | ISO/IEC 25010 | Atributos de calidad como guía de diseño: fiabilidad, mantenibilidad, seguridad, rendimiento |
| Arquitectura | IEEE/ISO 42010 | Descripción de vistas y puntos de vista arquitectónicos en la documentación de sistemas |
| Arquitectura | ISO/IEC/IEEE 15288 | Procesos del ciclo de vida de sistemas como referencia en proyectos complejos |
| Ciclo de Vida SW | ISO/IEC/IEEE 12207 | Procesos de desarrollo, mantenimiento y operación de software |
| Seguridad | ISO 27001 | Principios de gestión de seguridad de la información aplicados al diseño de sistemas |
| Seguridad | OWASP Top 10 | Mitigación de vulnerabilidades críticas en APIs y aplicaciones web |
| Seguridad | NIST CSF | Marco de ciberseguridad como referencia en análisis de riesgos y controles |
